📖 專案概述

什麼是 TestComplete 測試專案？

本專案是基於 TestComplete 的 Python 測試腳本，提供購物車功能的自動化測試，並包含完整的轉換指南，協助您將測試遷移到 FlaUI BDD 架構。

▶️ 執行測試

🖥️ 方法一：使用 TestComplete IDE

  1. 開啟 TestComplete
  2. 載入專案：Testcomplete_testcase/Testcase_shopping_cart
  3. 在 Project Explorer 中選擇要執行的測試
  4. 點擊工具列的 Run 按鈕
  5. 測試執行完成後，在 Test Log 中查看結果
💡 提示： 確保測試目標應用程式（購物車網頁）已正確啟動並可訪問。

⚙️ 方法二：使用命令列

# 執行整個專案 TestComplete.exe "YourProject.pjs" /r /project:"Testcase_shopping_cart" # 執行特定測試 TestComplete.exe "YourProject.pjs" /r /test:"TestItem1" # 匯出 JUnit 報告 TestComplete.exe "YourProject.pjs" /r /ExportSummary:"reports\junit-results.xml"

🏷️ 方法三：使用 Tag 執行

# 執行特定 tag 的測試 TestComplete.exe "YourProject.pjs" /r /tag:"@ShoppingCart" # 使用 tag expression TestComplete.exe "YourProject.pjs" /r /tag:"@AddItem and @Priority1"
⚠️ 注意： 執行測試前請確保：
  • 測試應用程式（購物車網頁）可正常訪問
  • TestComplete 已正確安裝並啟動
  • 專案路徑和設定正確

📊 產生測試報告

1 匯出 JUnit 報告

方法 A：使用 TestComplete UI

  1. 執行測試完成後，開啟 Summary Report
  2. 點擊工具列的 Export Summary as JUnit
  3. 儲存為：Project_Testcomplete/reports/junit-results.xml

方法 B：在測試腳本中自動匯出

# 在測試結尾加入 Log.SaveResultsAs( r"C:\path\to\SQA_AI_Automation\Project_Testcomplete\reports\junit-results.xml", lsJUnit )

方法 C：使用命令列參數

TestComplete.exe YourProject.pjs /r /ExportSummary:"...\reports\junit-results.xml"

執行 PowerShell 腳本來生成 Markdown 和 HTML 格式的摘要報告：

# 切換到專案目錄 cd Project_Testcomplete # 執行報告生成腳本 .\generate_report.ps1

🔄 轉換到 FlaUI BDD

為什麼要轉換？

將 TestComplete Python 測試轉換為 FlaUI BDD 可以獲得：

  • ✅ 更好的可讀性和維護性（使用 Gherkin 語法）
  • ✅ 更靈活的測試架構（Page Object Model）
  • ✅ 更好的整合能力（CI/CD 友好）
  • ✅ 開源免費的測試框架

1 轉換 Feature File

使用 Prompt 範本

參考 Transfer_Prompt.md 中的 Feature File 轉換 Prompt：

請將以下 TestComplete Python 測試案例轉換為 SpecFlow Feature file (.feature)： 要求： - 使用 Gherkin 語法 (Given, When, Then) - 每個 testcase_ 函數轉換為一個 Scenario - 測試步驟使用自然語言描述 - 包含 Feature 標題和描述 - 使用繁體中文或英文（根據需求） - 加入 @tags 標籤分類測試案例 輸入程式碼： [貼上 TestComplete Python 測試案例]

轉換範例

輸入 (TestComplete Python)：

def testcase_add_single_item(): """TC01：加入單一商品，驗證件數與總計。""" control_reset_cart() control_click_testid("add-apple") control_verify_text("cart-count", "1 件", "Cart count") control_verify_text("cart-total", "NT$ 30", "Cart total")

輸出 (SpecFlow Feature)：

Feature: 購物車功能測試 作為一個使用者 我想要測試購物車功能 以確保商品管理運作正常 @ShoppingCart @AddItem Scenario: TC01 - 加入單一商品 Given 購物車已清空 When 我點擊加入蘋果按鈕 Then 購物車件數應該是 "1 件" And 購物車總計應該是 "NT$ 30"

🔧 疑難排解

❌ 常見問題

問題 1：無法找到測試元素

症狀： 測試執行時提示找不到元素

解決方案：

  • 確認測試應用程式已正確啟動
  • 檢查元素的 AutomationId 或 Name 是否正確
  • 增加等待時間，確保元素已載入
  • 使用 TestComplete 的 Object Spy 重新檢查元素屬性

問題 2：JUnit 報告未正確匯出

症狀： 執行 generate_report.ps1 時提示找不到 junit-results.xml

解決方案：

  • 確認測試執行完成後已匯出 JUnit 報告
  • 檢查檔案路徑是否正確：reports/junit-results.xml
  • 確認有寫入權限到 reports 目錄
  • 使用 TestComplete UI 手動匯出報告

問題 3：轉換後的 FlaUI 測試無法執行

症狀： 轉換到 FlaUI 後測試失敗

解決方案：

  • 檢查元素定位策略是否正確轉換
  • 確認使用的是 FlaUI API 而非 TestComplete API
  • 驗證應用程式的 UI Automation 支援
  • 參考 Transfer_Prompt_Readme.md 中的 API 對照表
  • 逐步測試每個 Step Definition 確保功能正確

問題 4：PowerShell 腳本執行權限問題

症狀： 無法執行 .ps1 腳本

解決方案：

# 以管理員身份開啟 PowerShell，執行： Set-ExecutionPolicy RemoteSigned -Scope CurrentUser # 或者直接執行： powershell -ExecutionPolicy Bypass -File .\generate_report.ps1
