概述

Semi Inspection Desktop 為 WinForms 桌面程式，工作區根目錄預設為 Recipe_data。左側為檔案樹，右側為 RawData / Defect Chart 工作區與日誌區。

主要流程：Import Recipe → RawData → Defect Chart → Run Inspection（模擬 AOI 檢測）。

快速開始

步驟 1：建置

build_semi.bat

產出 SemiInspectionDesktop\bin\Debug\SemiInspectionDesktop.exe

步驟 2：啟動

run_semi.bat

未建置時會自動執行 build_semi.bat。

步驟 3：載入 Recipe

工具列 Import Recipe（Ctrl+I），或雙擊 File Tree 中的 InspectionRecipe_Sample.json。

步驟 4：檢視與分析

  1. RawData（Ctrl+E）— 檢視 Lot/Wafer、OpticalMode、三大 Policy
  2. Defect Chart（Ctrl+D）— DefectType / DefectCount 曲線圖
  3. Run Inspection（Ctrl+R）— 模擬檢測 log

預設 Recipe_data：demo2_desktop_app\Recipe_data

Recipe JSON 格式

{ "RecipeName": "Layer1_AOI_Recipe_v1", "WaferID": "W-20260605-001", "LotID": "LOT-SEM-A01", "Layer": "M1", "Step": "POST-ETCH", "ToolID": "AOI-TOOL-03", "OpticalMode": { "Illumination": "Brightfield", "PixelSize": 0.5 }, "DetectionPolicy": { "Algorithm": "PatternCompare", "Threshold": 45 }, "ReportingPolicy": { "MaxNuisanceRate": 5.0 }, "InspectionPolicy": { "ZoneOfInterest": "DieCore" }, "DefectSummary": [ { "DefectType": "Particle", "DefectCount": 18, "Severity": "Major", "DOI": true }, { "DefectType": "Scratch", "DefectCount": 6, "Severity": "Critical", "DOI": true } ] }

樣本：Recipe_data\InspectionRecipe_Sample.json（含 5 類 DefectSummary）

參數說明：SemiInspectionDesktop UIUX設定\INSPECTION_PARAMETERS.md

Defect Chart

需 Recipe 內含 DefectSummary 陣列。X 軸 = DefectType，Y 軸 = DefectCount。

按 Defect Chart 時會自動載入 Recipe（若尚未載入），並從 JSON 解析 DefectSummary 繪圖。

Sample Recipe 預期顯示 5 類缺陷：Particle(18)、Scratch(6)、Bridge(3)、Pattern(9)、Void(4)。

疑難排解

Import Recipe 失敗

  • 確認副檔名為 .json，且 JSON 格式正確
  • 檔案已在 Recipe_data 內時會直接載入

Defect Chart 無資料

  • 確認 JSON 含 DefectSummary 陣列
  • 先 Import Recipe 或按 RawData 載入樣本後再按 Defect Chart
  • 每筆需有 DefectType 與 DefectCount

run_semi.bat 無法執行

  • 確認 build_semi.bat 存在於同目錄
  • 關閉執行中的 SemiInspectionDesktop.exe 後重新 build
