📖 專案概述

什麼是 TestComplete 測試專案？

本專案是基於 TestComplete 的 Python 測試腳本，提供購物車功能的自動化測試，並包含完整的轉換指南，協助您將測試遷移到 FlaUI BDD 架構。

🏗️ 專案架構

  1. 開啟 TestComplete
  2. 載入專案：Testcomplete_testcase/Testcase_shopping_cart
  3. 在 Project Explorer 中選擇要執行的測試

🔄 轉換到 FlaUI BDD

為什麼要轉換？

將 TestComplete Python 測試轉換為 FlaUI BDD 可以獲得：

  • ✅ 更好的可讀性和維護性（使用 Gherkin 語法）
  • ✅ 更靈活的測試架構（Page Object Model）
  • ✅ 更好的整合能力（CI/CD 友好）
  • ✅ 開源免費的測試框架

1 轉換 Feature File

使用 Prompt 範本

請將以下 TestComplete Python 測試案例轉換為 SpecFlow Feature file (.feature)： 要求： - 使用 Gherkin 語法 (Given, When, Then) - 每個 testcase_ 函數轉換為一個 Scenario - 測試步驟使用自然語言描述 - 包含 Feature 標題和描述 - 使用繁體中文或英文（根據需求） - 加入 @tags 標籤分類測試案例 輸入程式碼： [貼上 TestComplete Python 測試案例]

3 建立 Page Object Model

將相關的 control_ 函數群組轉換為 Page Object 類別：

請將 control_ 函數群組轉換為 Page Object Model 類別： 要求： - 每個頁面/元件建立獨立的 Page Object 類別 - 封裝元素定位器 (Locators) - 封裝操作方法 (Actions) - 使用 FlaUI API - 實作 IDisposable 介面進行資源管理 輸入程式碼： [貼上 control_ 函數群組]

Page Object 架構設計

建議的 Page Object 結構：

  • BasePage.cs - 基礎類別，提供共用的元素查找和操作方法
  • ProductListPage.cs - 產品列表頁面物件
  • ShoppingCartPage.cs - 購物車頁面物件

範例 1：BasePage 基礎類別

目的：提供所有 Page Object 共用的基礎功能

using FlaUI.Core; using FlaUI.Core.AutomationElements; using FlaUI.Core.Tools; using FlaUI.UIA3; namespace ShoppingCartTests.PageObjects { /// <summary> /// 基礎頁面類別，提供所有頁面共用的功能 /// </summary> public abstract class BasePage : IDisposable { protected readonly Window Window; protected readonly UIA3Automation Automation; private readonly int DefaultTimeout = 10000; // 毫秒 protected BasePage(Window window, UIA3Automation automation) { Window = window ?? throw new ArgumentNullException(nameof(window)); Automation = automation ?? throw new ArgumentNullException(nameof(automation)); } /// <summary> /// 根據 AutomationId 尋找元素 /// </summary> protected AutomationElement FindElement(string automationId, int timeout = 0) { if (timeout == 0) timeout = DefaultTimeout; var element = Retry.WhileNull( () => Window.FindFirstDescendant(cf => cf.ByAutomationId(automationId)), TimeSpan.FromMilliseconds(timeout) ).Result; if (element == null) throw new Exception($"找不到元素: {automationId}"); return element; } /// <summary> /// 點擊元素 /// </summary> protected void ClickElement(string automationId) { var element = FindElement(automationId); element.AsButton()?.Click(); } /// <summary> /// 取得元素文字 /// </summary> protected string GetElementText(string automationId) { var element = FindElement(automationId); return element.Name ?? element.Properties.Name.ValueOrDefault ?? string.Empty; } /// <summary> /// 等待元素出現 /// </summary> protected bool WaitForElement(string automationId, int timeout = 5000) { try { FindElement(automationId, timeout); return true; } catch { return false; } } public void Dispose() { // 清理資源 } } }

using FlaUI.Core.AutomationElements; using FlaUI.UIA3; namespace ShoppingCartTests.PageObjects { /// <summary> /// 購物車頁面物件 /// </summary> public class ShoppingCartPage : BasePage { // 元素定位器 private const string CartCountId = "cart-count"; private const string CartTotalId = "cart-total"; private const string ResetCartButtonId = "reset-cart"; private const string CheckoutButtonId = "checkout"; public ShoppingCartPage(Window window, UIA3Automation automation) : base(window, automation) { } /// <summary> /// 取得購物車件數 /// </summary> public string GetCartCount() { return GetElementText(CartCountId); } /// <summary> /// 取得購物車總計 /// </summary> public string GetCartTotal() { return GetElementText(CartTotalId); } /// <summary> /// 點擊增加數量按鈕 /// </summary> public void ClickIncreaseQuantity(string itemName) { var buttonId = $"increase-{GetItemKey(itemName)}"; ClickElement(buttonId); } /// <summary> /// 點擊移除商品按鈕 /// </summary> public void ClickRemoveItem(string itemName) { var buttonId = $"remove-{GetItemKey(itemName)}"; ClickElement(buttonId); } /// <summary> /// 清空購物車 /// </summary> public void ResetCart() { ClickElement(ResetCartButtonId); } /// <summary> /// 點擊結帳按鈕 /// </summary> public void ClickCheckout() { ClickElement(CheckoutButtonId); } /// <summary> /// 將中文商品名轉為 ID key /// </summary> private string GetItemKey(string itemName) { return itemName switch { "蘋果" => "apple", "香蕉" => "banana", "牛奶" => "milk", _ => itemName.ToLower() }; } } }

⚠️ 常見轉換陷阱

❌ 陷阱 1：直接複製 XPath

問題：TestComplete 的 XPath 在 FlaUI 中不適用

解決：改用 AutomationId、Name 或 ClassName 定位

// ❌ 錯誤 element.FindByXPath("//*[@data-testid='cart-count']") // ✅ 正確 element.FindFirstDescendant(cf => cf.ByAutomationId("cart-count"))

❌ 陷阱 2：忽略等待時間

問題：元素尚未載入就嘗試操作

解決：使用 Retry.WhileNull 或明確等待

// ✅ 使用重試機制 var element = Retry.WhileNull( () => Window.FindFirstDescendant(cf => cf.ByAutomationId(id)), TimeSpan.FromSeconds(10) ).Result;

❌ 陷阱 3：過度耦合的 Step Definitions

問題：Step Definition 中包含太多 UI 邏輯

解決：將 UI 操作封裝到 Page Object

// ❌ 錯誤 - Step 中有太多細節 [When(@"我點擊加入蘋果")] public void When我點擊加入蘋果() { var btn = Window.FindFirstDescendant(cf => cf.ByAutomationId("btn-add-apple")); btn.AsButton().Click(); } // ✅ 正確 - 使用 Page Object [When(@"我點擊加入蘋果")] public void When我點擊加入蘋果() { _productListPage.ClickAddApple(); }

❌ 陷阱 4：未處理異常

問題：測試失敗時缺乏診斷資訊

解決：加入 try-catch 和截圖機制

[Then(@"購物車件數應該是 ""(.*)""")] public void Then購物車件數應該是(string expected) { try { var actual = _shoppingCartPage.GetCartCount(); Assert.That(actual, Is.EqualTo(expected)); TestContext.WriteLine($"✅ 驗證成功: {actual}"); } catch (Exception ex) { ScreenshotHelper.TakeScreenshot(_scenarioContext, "CartCount_Failed"); TestContext.WriteLine($"❌ 驗證失敗: {ex.Message}"); throw; } }

🔧 疑難排解

❌ 常見問題

問題 1：無法找到測試元素

症狀： 測試執行時提示找不到元素

解決方案：

  • 確認測試應用程式已正確啟動
  • 檢查元素的 AutomationId 或 Name 是否正確
  • 增加等待時間，確保元素已載入
  • 使用 TestComplete 的 Object Spy 重新檢查元素屬性

問題 2：JUnit 報告未正確匯出

症狀： 執行 generate_report.ps1 時提示找不到 junit-results.xml

解決方案：

  • 確認測試執行完成後已匯出 JUnit 報告
  • 檢查檔案路徑是否正確：reports/junit-results.xml
  • 確認有寫入權限到 reports 目錄
  • 使用 TestComplete UI 手動匯出報告

問題 3：轉換後的 FlaUI 測試無法執行

症狀： 轉換到 FlaUI 後測試失敗

解決方案：

  • 檢查元素定位策略是否正確轉換
  • 確認使用的是 FlaUI API 而非 TestComplete API
  • 驗證應用程式的 UI Automation 支援
  • 參考 Transfer_Prompt_Readme.md 中的 API 對照表
  • 逐步測試每個 Step Definition 確保功能正確

問題 4：PowerShell 腳本執行權限問題

症狀： 無法執行 .ps1 腳本

解決方案：

# 以管理員身份開啟 PowerShell，執行： Set-ExecutionPolicy RemoteSigned -Scope CurrentUser # 或者直接執行： powershell -ExecutionPolicy Bypass -File .\generate_report.ps1
