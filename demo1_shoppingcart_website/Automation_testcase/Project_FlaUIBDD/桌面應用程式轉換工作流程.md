# Desktop Application：TestComplete → FlaUI BDD 完整轉換工作流程

本文件涵蓋將 **桌面應用程式 TestComplete 測試專案**遷移至 **FlaUI BDD（SpecFlow + NUnit + FlaUI）** 的端對端工作流程，包含環境建構、程式碼轉換、報告產生及 Bamboo CI/CD 整合。

---

## 📋 工作流程總覽

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1：建構 FlaUI + Inspector 環境                         │
│  Step 2：TestComplete → FlaUI BDD 轉換                       │
│    2-1  Testcases  → BDD Feature Files                      │
│    2-2  Modules    → BDD StepDefinitions                    │
│    2-3  Name Mapping → PageObjects / Element Objects        │
│  Step 3：Result Report 整合                                  │
│  Step 4：建構 Bamboo CI/CD Plan                              │
└─────────────────────────────────────────────────────────────┘
```

## Step 1：建構 FlaUI + Inspector 環境

### 1-1. 前置需求（Build Agent / 開發機）

| 項目 | 版本 / 說明 |
|------|------------|
| OS | **Windows 10/11** 或 **Windows Server 2019/2022**（必須為 Windows） |
| .NET SDK | .NET 8.0 SDK（[下載](https://dotnet.microsoft.com/download/dotnet/8.0)） |
| Visual Studio | VS 2022 Community+（可選，含 SpecFlow for VS 擴充功能） |
| Git | 最新版 |
| Python | 3.9+（執行 demo server 或報告產生腳本） |

> ⚠️ FlaUI 透過 Windows UI Automation (UIA3) 操作桌面應用，**不支援 Linux / macOS**。

---

### 1-2. 安裝 FlaUI NuGet 套件

在 `.csproj` 中加入以下套件（或使用 `dotnet add package`）：

```xml
<!-- FlaUI 核心套件 -->
<PackageReference Include="FlaUI.Core" Version="4.*" />
<PackageReference Include="FlaUI.UIA3" Version="4.*" />

<!-- BDD 框架 -->
<PackageReference Include="SpecFlow" Version="3.*" />
<PackageReference Include="SpecFlow.NUnit" Version="3.*" />
<PackageReference Include="SpecFlow.Tools.MsBuild.Generation" Version="3.*" />

<!-- 測試執行 -->
<PackageReference Include="NUnit" Version="3.*" />
<PackageReference Include="NUnit3TestAdapter" Version="4.*" />
<PackageReference Include="Microsoft.NET.Test.Sdk" Version="17.*" />

<!-- 報告（可選） -->
<PackageReference Include="ExtentReports" Version="4.*" />
<PackageReference Include="NUnit.Extension.NUnitV2ResultWriter" Version="3.*" />
```

安裝指令：
```powershell
dotnet add package FlaUI.Core
dotnet add package FlaUI.UIA3
dotnet add package SpecFlow.NUnit
dotnet add package ExtentReports
```

### 1-3. 安裝 FlaUI Inspector（元素辨識工具）

FlaUI Inspector 是取代 TestComplete **Object Spy / Name Mapping** 的視覺化元素瀏覽工具。

**安裝方式**：
1. 前往 [FlaUI Inspector GitHub Releases](https://github.com/FlaUI/FlaUIInspector/releases)
2. 下載最新 `FlaUIInspector.zip`，解壓縮後直接執行 `FlaUIInspector.exe`

**主要功能**：

| 功能 | 說明 |
|------|------|
| 元素樹狀瀏覽 | 與 TestComplete Object Browser 相同概念 |
| AutomationId 查詢 | 取得元素的 `AutomationId`（最優先使用） |
| Name / ClassName 查詢 | 備用定位屬性 |
| ControlType 識別 | Button / Edit / ListItem / TreeItem 等 |
| XPath 產生 | 自動產生 FlaUI 相容的 XPath |
| Hover 模式 | 滑鼠停留即高亮顯示元素資訊 |

**輔助工具**：
- **Accessibility Insights for Windows**（Microsoft 官方，[下載](https://accessibilityinsights.io/docs/windows/overview/)）
- **Inspect.exe**（Windows SDK 內建，位於 `C:\Program Files (x86)\Windows Kits\10\bin\<version>\x64\`）

### 1-4. 建立 FlaUI BDD 專案結構

```
MyApp_FlaUI_BDD/
├── Features/                   # Gherkin 測試場景（.feature 檔）
├── StepDefinitions/            # Given/When/Then 步驟實作（C#）
├── PageObjects/                # 頁面/元件物件封裝（對應 Name Mapping）
│   ├── BasePage.cs             # 基礎類別：應用程式啟動、通用等待
│   ├── MainWindowPage.cs       # 主視窗元素與操作
│   └── [其他功能頁面].cs
├── Hooks/
│   └── TestHooks.cs            # BeforeScenario / AfterScenario 生命週期
├── Helpers/
│   ├── ConfigHelper.cs         # 讀取 App.config / specflow.json
│   ├── ScreenshotHelper.cs     # 失敗自動截圖
│   └── HtmlReportHelper.cs     # ExtentReports 整合
├── reports/                    # 測試輸出目錄
├── App.config                  # 應用程式路徑、等待逾時等設定
├── specflow.json               # SpecFlow 設定
└── MyApp_FlaUI_BDD.csproj
```

**specflow.json 基本設定**：
```json
{
  "$schema": "https://specflow.org/json-schemas/specflow.json",
  "language": {
    "feature": "zh-TW"
  },
  "bindingCulture": {
    "name": "zh-TW"
  }
}
```

**App.config 應用程式路徑設定**：
```xml
<appSettings>
  <add key="AppPath" value="C:\Program Files\MyApp\MyApp.exe" />
  <add key="AppWorkDir" value="C:\Program Files\MyApp\" />
  <add key="WaitTimeout" value="10" />
  <add key="ScreenshotOnFailure" value="true" />
</appSettings>
```

## Step 2：TestComplete → FlaUI BDD 轉換

### 轉換對照總表

| TestComplete 結構 | 對應 FlaUI BDD 結構 |
|------------------|-------------------|
| `testcase_xxx()` 函數 | `.feature` 中的 `Scenario` |
| `control_xxx()` 函數 | `StepDefinitions` 中的 `[Given]` / `[When]` / `[Then]` |
| `_aliases.py` Name Mapping | `PageObjects/*.cs` 元素封裝 |
| `Log.Error()` / `Log.Checkpoint()` | `Assert.AreEqual()` / `Assert.That()` |
| Object Browser 路徑 | `AutomationId` / `Name` / `ClassName` |

---

### 2-1. TestComplete Testcases → BDD Feature Files

**轉換策略**：
- 每個 `testcase_` 函數 → 一個 `Scenario`
- 有多組測試資料的 → 改用 `Scenario Outline + Examples`
- 測試案例 ID（如 TC01）保留在 Scenario 標題或 `@tag` 中

**轉換步驟**：

1. **分析 testcase_ 函數**，整理：
   - 測試目的（Scenario 標題）
   - 前置條件（`Given` 步驟）
   - 執行動作（`When` 步驟）
   - 驗證點（`Then` / `And` 步驟）

2. **撰寫 Feature File**（Gherkin）

3. **使用 AI 輔助轉換 Prompt**：
```
請將以下 TestComplete Python testcase_ 函數轉換為 SpecFlow .feature 檔案：

要求：
- 使用 Gherkin 語法（Given / When / Then / And）
- 每個 testcase_ 函數對應一個 Scenario
- 若有多組輸入資料，使用 Scenario Outline + Examples
- 保留原有測試案例 ID（TC01, TC02...）作為 @tag 或標題
- 繁體中文描述
- 加入功能分類 @tag

[貼入 testcase_ 函數程式碼]
```

**轉換範例**：

```python
# TestComplete Python（輸入）
def testcase_add_single_item():
    """TC01：加入單一商品"""
    control_launch_app()
    control_click_button("add-apple")
    control_verify_label("cart-count", "1 件")
```

```gherkin
# SpecFlow Feature（輸出）
@TC01 @AddItem
Scenario: TC01 - 加入單一商品
  Given 應用程式已啟動
  When 使用者點擊「加入蘋果」按鈕
  Then 購物車件數應顯示 "1 件"
```

---

### 2-2. TestComplete Modules → BDD StepDefinitions

**轉換策略**：
- `control_click_xxx()` → `[When(@"使用者點擊...")]`
- `control_verify_xxx()` → `[Then(@"...應顯示...")]`
- `control_launch_app()` → `[Given(@"應用程式已啟動")]`（通常移至 Hooks）
- 共用狀態透過 `ScenarioContext` 傳遞

**轉換步驟**：

1. 列出所有 `control_` 函數，分類為：
   - 啟動/關閉應用程式 → `Hooks.cs`
   - 點擊/輸入操作 → `[When]` 步驟
   - 驗證/斷言 → `[Then]` 步驟
   - 取得資料 → `[When]` 步驟（存入 ScenarioContext）

2. **使用 AI 輔助轉換 Prompt**：
```
請將以下 TestComplete Python control_ 函數轉換為 SpecFlow StepDefinitions（C#）：

要求：
- 使用 FlaUI UIA3 API
- 元素定位優先使用 AutomationId，其次 Name，最後 ClassName
- 使用 ScenarioContext 在步驟間共享資料
- 加入 WaitUntilEnabled() / WaitUntilResponsive() 等待機制
- Assert 使用 NUnit 的 Assert.That() 並加入明確錯誤訊息
- 失敗時呼叫 ScreenshotHelper.TakeScreenshot()

[貼入 control_ 函數程式碼]
```

**FlaUI API 對照**：

| TestComplete | FlaUI C# |
|-------------|----------|
| `Sys.Process("MyApp")` | `Application.Launch(@"C:\...\MyApp.exe")` |
| `element.Click()` | `button.Click()` |
| `element.SetText("abc")` | `textBox.Enter("abc")` |
| `element.contentText` | `element.AsLabel().Text` 或 `element.Name` |
| `element.Visible` | `element.IsEnabled` |
| `page.WaitProperty("Exists", true)` | `Retry.WhileNull(() => element, TimeSpan.FromSeconds(10))` |
| `Log.Checkpoint("Pass", msg)` | `Assert.That(actual, Is.EqualTo(expected), msg)` |
| `aqObject.CheckProperty(el, "Text", val)` | `Assert.That(element.Name, Is.EqualTo(val))` |

**StepDefinitions 範本**：
```csharp
[Binding]
public class AppSteps
{
    private readonly ScenarioContext _scenarioContext;
    private readonly BasePage _basePage;

    public AppSteps(ScenarioContext scenarioContext, BasePage basePage)
    {
        _scenarioContext = scenarioContext;
        _basePage = basePage;
    }

    [When(@"使用者點擊「(.*)」按鈕")]
    public void WhenUserClicksButton(string buttonName)
    {
        var button = _basePage.MainWindow
            .FindFirstDescendant(cf => cf.ByName(buttonName))
            ?.AsButton();
        button.Should().NotBeNull($"找不到按鈕：{buttonName}");
        button.Click();
    }

    [Then(@"購物車件數應顯示 ""(.*)""")]
    public void ThenCartCountShouldBe(string expected)
    {
        var label = _basePage.MainWindow
            .FindFirstDescendant(cf => cf.ByAutomationId("cart-count"))
            ?.AsLabel();
        Assert.That(label?.Text, Is.EqualTo(expected), "購物車件數不符");
    }
}
```

### 2-3. TestComplete Name Mapping → FlaUI PageObjects

**TestComplete Name Mapping 結構說明**：
- `_aliases.py` 或 Name Mapping 編輯器儲存元素別名（Alias）
- 路徑通常為 `Aliases.MyApp.MainWindow.Panel.BtnAdd`
- 元素屬性包含 `ObjectType`, `WndClass`, `WndCaption`, `ObjectIdentifier`

**轉換策略**：

1. **開啟 FlaUI Inspector**，啟動受測應用程式
2. **對應每個 Name Mapping 別名**，找到對應的：
   - `AutomationId`（最優先）
   - `Name`（視窗標題、控制項名稱）
   - `ClassName`（WndClass 對應）
   - `ControlType`（Button, Edit, List 等）
3. **建立 PageObject 類別**，每個主要頁面/視窗一個類別
4. 元素定義為 **Lazy Property**（延遲初始化）

**Name Mapping 對照轉換範例**：

```python
# TestComplete _aliases.py（輸入）
# Aliases.MyApp.MainWindow.BtnAddApple  (ObjectIdentifier="add-apple")
# Aliases.MyApp.MainWindow.LblCartCount (ObjectIdentifier="cart-count")
# Aliases.MyApp.CartDialog.LstCartItems (WndClass="ListBox")
```

```csharp
// FlaUI PageObject（輸出）
public class MainWindowPage : BasePage
{
    // 對應 ObjectIdentifier / AutomationId
    public Button BtnAddApple =>
        MainWindow.FindFirstDescendant(cf => cf.ByAutomationId("add-apple"))?.AsButton();

    public Label LblCartCount =>
        MainWindow.FindFirstDescendant(cf => cf.ByAutomationId("cart-count"))?.AsLabel();
}

public class CartDialogPage : BasePage
{
    // 對應 WndClass → ClassName
    public ListBox LstCartItems =>
        MainWindow.FindFirstDescendant(cf => cf.ByClassName("ListBox"))?.AsListBox();
}
```

**使用 AI 輔助轉換 Prompt**：
```
請將以下 TestComplete Name Mapping / _aliases 定義轉換為 FlaUI PageObject 類別（C#）：

要求：
- 每個主要視窗/頁面建立獨立的 PageObject 類別
- 元素以 Lazy Property（get accessor）定義，每次呼叫重新查找
- 定位策略優先順序：ByAutomationId > ByName > ByClassName > ByControlType
- 若元素在子容器內，使用父容器進行 FindFirstDescendant 縮小範圍
- 繼承 BasePage 基礎類別
- 加入 XML 文件注釋說明對應的原始 Alias 路徑

[貼入 _aliases.py 或 Name Mapping 匯出內容]
```

**元素定位策略對照**：

| TestComplete 屬性 | FlaUI 定位方式 |
|-----------------|--------------|
| `ObjectIdentifier` / `data-testid` | `ByAutomationId("id")` |
| `WndCaption` / 視窗標題 | `ByName("標題")` |
| `WndClass` | `ByClassName("ClassName")` |
| `ObjectType = "Button"` | `ByControlType(ControlType.Button)` |
| XPath 路徑 | `FindFirstByXPath("//*[@AutomationId='id']")` |
| 巢狀路徑 A > B > C | `panelA.FindFirstDescendant(cf => cf.ByAutomationId("C"))` |

## Step 3：Result Report 整合

### 3-1. 測試輸出格式

FlaUI BDD 專案支援以下報告格式：

| 格式 | 工具 | 用途 |
|------|------|------|
| **JUnit XML** | NUnit3TestAdapter | Bamboo / Jenkins 解析 |
| **HTML Report** | ExtentReports | 人工閱讀、截圖嵌入 |
| **NUnit XML** | NUnit | 詳細測試結果 |
| **Summary Report** | 自訂 PowerShell 腳本 | 管理層摘要 |

---

### 3-2. 設定 JUnit XML 輸出

執行測試時加入 logger 參數：
```powershell
dotnet test `
  --logger "junit;LogFilePath=reports/junit-results.xml" `
  --logger "html;LogFileName=reports/TestReport.html"
```

或在 `run-tests-and-report.ps1` 中統一管理：
```powershell
dotnet test `
  --configuration $Configuration `
  --logger "junit;LogFilePath=$ReportDir\junit-results.xml" `
  --logger "nunit;LogFilePath=$ReportDir\nunit-results.xml" `
  --results-directory $ReportDir
```

---

### 3-3. ExtentReports HTML 報告

在 `Hooks/TestHooks.cs` 中整合 ExtentReports：

```csharp
[BeforeTestRun]
public static void BeforeTestRun()
{
    HtmlReportHelper.Initialize("reports/TestReport.html");
}

[AfterScenario]
public void AfterScenario()
{
    if (_scenarioContext.TestError != null)
    {
        // 失敗截圖
        var screenshotPath = ScreenshotHelper.TakeScreenshot(
            $"reports/screenshots/{_scenarioContext.ScenarioInfo.Title}");
        HtmlReportHelper.LogFailure(_scenarioContext, screenshotPath);
    }
    else
    {
        HtmlReportHelper.LogPass(_scenarioContext);
    }
}

[AfterTestRun]
public static void AfterTestRun()
{
    HtmlReportHelper.Flush();
}
```

---

### 3-4. Summary Report（管理層摘要）

使用 `generate_report.ps1` 從 JUnit XML 產生 Markdown / HTML 摘要報告：

```powershell
# 執行測試並產生完整報告
.\run-tests-and-report.ps1

# 僅從已有的 junit-results.xml 產生報告
.\generate_report.ps1
```

報告輸出位置：
```
reports/
├── junit-results.xml       # CI 解析用（JUnit 格式）
├── TestReport.html         # ExtentReports HTML（含截圖）
├── summary_report.md       # Markdown 摘要
└── summary_report.html     # HTML 摘要（瀏覽器開啟）
```

## Step 4：建構 Bamboo CI/CD Plan

### 4-1. Bamboo Agent 需求

| 項目 | 需求 |
|------|------|
| OS | **Windows Server 2019/2022** 或 **Windows 10/11**（必須） |
| Agent 類型 | Remote Agent（建議）或 Local Agent |
| .NET SDK | .NET 8.0 SDK 已安裝 |
| 受測應用程式 | 已安裝，或可在 Build Task 中自動安裝 |
| 螢幕顯示 | **需要真實桌面 Session** 或 Virtual Display（如 VNC）<br>建議以互動模式登入 Agent 服務（Interactive Session） |
| Agent Capability | 加入 `custom.flaui.enabled=true` 標籤以篩選 Agent |

> ⚠️ **重要**：FlaUI 操控 Windows UI 需要桌面存取權限。Bamboo Agent 服務帳號必須能夠建立視窗 Session（避免以 System 帳號執行服務）。

---

### 4-2. Bamboo Plan 結構

```
Plan: MyApp - FlaUI BDD Regression
│
├── Stage 1: Build & Prepare
│   └── Job: Build
│       ├── Task 1: Source Code Checkout
│       ├── Task 2: dotnet restore
│       └── Task 3: dotnet build
│
├── Stage 2: Test Execution
│   └── Job: FlaUI Tests
│       ├── Task 1: Launch Application（可選，若需自動啟動受測 APP）
│       ├── Task 2: dotnet test（含 JUnit logger）
│       └── Task 3: Stop Application（可選）
│
└── Stage 3: Report & Publish
    └── Job: Report
        ├── Task 1: generate_report.ps1
        ├── Task 2: Artifact 發布（HTML / XML 報告）
        └── Task 3: JUnit Parser（解析測試結果）
```

---

### 4-3. Bamboo 各 Task 設定

#### Task 1：Source Code Checkout
```yaml
Type: Source Code Checkout
Repository: [Git repository 名稱]
Checkout Directory: (留空，使用預設)
```

#### Task 2：dotnet restore
```yaml
Type: Script（PowerShell）
Script body: |
  dotnet restore Project_FlaUIBDD\Testcase_shopping_cart_FlaUI_BDD\MyApp_FlaUI_BDD.csproj
```

#### Task 3：dotnet build
```yaml
Type: Script（PowerShell）
Script body: |
  dotnet build Project_FlaUIBDD\MyApp_FlaUI_BDD\ `
    --configuration Release `
    --no-restore
```

#### Task 4：dotnet test（執行 FlaUI 測試）
```yaml
Type: Script（PowerShell）
Script body: |
  $timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
  $reportDir = "Project_FlaUIBDD\MyApp_FlaUI_BDD\reports"
  
  dotnet test Project_FlaUIBDD\MyApp_FlaUI_BDD\ `
    --configuration Release `
    --no-build `
    --logger "junit;LogFilePath=$reportDir\junit-results.xml" `
    --logger "html;LogFileName=$reportDir\TestReport.html" `
    --results-directory $reportDir `
    -- NUnit.NumberOfTestWorkers=1
```

> 💡 桌面 UI 測試建議設定 `NUnit.NumberOfTestWorkers=1`，避免多執行緒同時操控同一 UI。

#### Task 5：產生 Summary Report
```yaml
Type: Script（PowerShell）
Script body: |
  cd Project_FlaUIBDD
  .\generate_report.ps1
Working directory: ${bamboo.build.working.directory}
```

#### Task 6：JUnit 測試結果解析
```yaml
Type: JUnit Parser
Specify custom results directories: |
  **/reports/junit-results.xml
```

#### Task 7：Artifact 發布
```yaml
Type: Artifact Definition（在 Plan 設定）

Artifact 1 - HTML Report:
  Name: FlaUI-HTML-Report
  Copy pattern: Project_FlaUIBDD/**/reports/TestReport.html
  Shared: false

Artifact 2 - Screenshots:
  Name: FlaUI-Screenshots
  Copy pattern: Project_FlaUIBDD/**/reports/screenshots/**
  Shared: false

Artifact 3 - JUnit XML:
  Name: FlaUI-JUnit-XML
  Copy pattern: Project_FlaUIBDD/**/reports/junit-results.xml
  Shared: true
```

### 4-4. Bamboo Plan 變數設定

在 Plan 層級設定以下變數：

| 變數名稱 | 預設值 | 說明 |
|---------|--------|------|
| `app.path` | `C:\Program Files\MyApp\MyApp.exe` | 受測應用程式路徑 |
| `test.config` | `Release` | Build 組態 |
| `test.filter` | `""` | 測試篩選（空=全跑，如 `@Smoke`） |
| `report.dir` | `reports` | 報告輸出目錄 |

在測試 Task 中引用：
```powershell
dotnet test ... `
  --filter "Category=$env:bamboo_test_filter"
```
